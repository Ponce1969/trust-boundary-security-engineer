# Purpose

Security engineering knowledge is scattered across books, standards, and tools. Most of it assumes the reader already knows where to look. This project exists to concentrate that knowledge into a portable, structured skill that any AI agent can adopt — without installing anything, running anything, or depending on a specific platform.

The goal is not to automate security. It is to teach it.

# Vision

Every AI agent that touches code, infrastructure, or systems should carry a baseline understanding of security engineering: how to think adversarially, how to reason about risk, and how to communicate findings responsibly.

This project aims to make that baseline available as plain documentation — readable by humans, loadable by agents, and independent of any framework or vendor.

The project favors explanation over automation and understanding over complexity.

# Goals

- Teach responsible Security Engineering practices through structured documentation.
- Help AI agents explain findings clearly and with evidence, not speculation.
- Encourage secure-by-default thinking in design and implementation decisions.
- Reduce cognitive load for developers and teams navigating security concerns.
- Prioritize observable evidence over assumptions in every finding and recommendation.
- Promote human oversight as a non-negotiable part of any security process.

# Non-Goals

- Not a vulnerability scanner, linter, or detection tool.
- Not a Python package, library, or installable module.
- Not an MCP server or network service.
- Not a framework requiring integration or configuration.
- Not a replacement for human judgment or professional review.

# Intended Audience

- Developers who want to write more secure code and understand why.
- AI agent users who expect grounded, responsible security guidance.
- Security-conscious teams looking for a shared reference they can extend.
- Open source contributors who value clarity and educational rigor.

# Success Criteria

- **Clarity.** Every document is understandable on first reading.
- **Simplicity.** No concept requires more explanation than necessary.
- **Reusability.** Any AI agent on any platform can adopt the skill without modification.
- **Platform independence.** The skill works wherever Markdown works.
- **Educational value.** A reader — human or agent — finishes each entry knowing something they did not know before.
